03 De qué va este proyecto

Una integración.
Todo el ecosistema financiero.

Propuesta de valor

Conecta una vez. Opera con todos.

Una plataforma de integración única para que empresas y fintechs enlacen con varios bancos por APIs estandarizadas: menos fricción técnica, pagos más simples y un camino más corto para lanzar, cobrar y escalar.

Problema que resuelve

  • Integraciones distintas por banco: más costo, más riesgo y equipos saturados.
  • Operación manual que no escala cuando crece el volumen.
  • Ciclos largos de implementación antes de ver ingresos reales.

PyMEs

  • Representan aproximadamente 97% del tejido empresarial en Panamá (más de 60,000 empresas).
  • Alta necesidad de digitalización y automatización de pagos.

Insight Segmento clave para escala y adopción masiva, aunque con menor ticket individual.

Empresas medianas y grandes

  • Representan menos del 5% del total, pero concentran el mayor volumen transaccional.

Insight Segmento crítico para generación de ingresos, especialmente en pagos masivos y recaudos.

Qué desbloqueas

  • Un solo frente tecnológico para integrar y mantener.
  • Acceso al universo bancario acordado, sin renegociar uno a uno.
  • Automatización y escala en pagos, recaudos y operación diaria.

Capacidades

  • Integración y conectividad
  • Pagos y transferencias
  • Cobros y adquirencia
  • Pagos masivos
  • Recurrentes (cuenta a cuenta)
  • Recaudos y cobranzas
  • Gestión y control
  • Seguridad y autorización
  • Operación simplificada

Ganancia para los bancos

Más transacciones y nuevos casos de uso sobre la confianza que ya tienen con su infraestructura: sin cambiar su rol y sin megaproyectos de IT cada vez que entra un aliado.

No sustituimos sus canales: los evolucionamos hacia un modelo más eficiente, interoperable y escalable. Ganan empresas, fintechs, bancos y el cliente final.

Alcance y límites

Alcance del Modelo

Naturaleza del modelo, alcance operativo incluido y excluido, y mensaje ejecutivo.

1. Naturaleza

El modelo corresponde a una infraestructura tecnológica de integración financiera (API-first) que habilita la conexión entre instituciones financieras y terceros.

Opera como una capa intermedia (middleware), bajo un esquema B2B2C, sin intervenir en la relación financiera entre el banco y el cliente final.

2. Alcance incluido

  • Iniciación de pagos cuenta-a-cuenta (A2A)
  • Transferencias interbancarias (ACH)
  • Débitos autorizados y request-to-pay
  • Pagos digitales (QR, links)
  • Payouts, nómina y recaudos
  • APIs, SDKs y orquestación de transacciones

3. Alcance excluido

  • Custodia de fondos o apertura de cuentas
  • Emisión de productos financieros
  • Procesos de cumplimiento (AML/KYC)
  • Autorización de transacciones
  • Relación directa con el cliente final

4. Mensaje clave

La solución actúa como habilitador tecnológico del ecosistema financiero, donde los bancos conservan el control de los fondos y las decisiones, mientras la plataforma estandariza e integra la operación entre participantes.

3. Características y Funcionalidades

API · AaaS

Segmentos, dolores y solución mediante API (Telered AaaS)

Primer bloque de segmentos prioritarios: actores que más rápido capturan eficiencia, cobertura y escalabilidad con una integración estandarizada.

01

Fintechs / Wallets / Plataformas de pago

Kash, Zinli, Mercado Pago, dLocal

Dolores

  • Integración compleja con múltiples bancos
  • Dependencia de acuerdos bilaterales
  • Acceso limitado a rails locales (ACH)
  • Dificultad para operar pagos locales en esquemas cross-border

Solución API

  • Integración única estandarizada
  • Acceso directo a pagos cuenta-a-cuenta (A2A)
  • Pagos locales para players globales como dLocal
  • Lanzamiento ágil de nuevos productos financieros

02

Payroll / BPO / Empresas de nómina

PayDay, empresas de outsourcing

Dolores

  • Procesos manuales de pago
  • Conciliación compleja
  • Riesgo operativo en pagos masivos

Solución API

  • Automatización de pagos masivos
  • Integración directa con cuentas bancarias
  • Confirmaciones y conciliación en tiempo real

03

Marketplaces / Plataformas digitales

Encuentra24, Degusta

Dolores

  • Dificultad para cobrar y pagar a terceros
  • Dependencia de efectivo o transferencias manuales
  • Falta de infraestructura de payouts

Solución API

  • Cobros digitales con QR, links y A2A
  • Pagos automatizados a usuarios
  • Escalabilidad operativa

Lectura estratégica

La tesis económica no depende de capturar todo el mercado, sino de concentrarse en el tramo accesible donde la API reduce fricción, acelera adopción y gana volumen.

El esquema debe privilegiar alta adopción y escalabilidad transaccional, manteniendo un costo competitivo frente a las tarjetas.

5. Modelo Operativo y Gobernanza

Operación

Modelo Operativo y de Gobernanza – Resumen Ejecutivo

La operación se sostiene sobre una infraestructura neutral donde Telered integra, el banco controla y la empresa origina la transacción.

Principio

Telered opera como una infraestructura tecnológica neutral, que habilita la conexión entre bancos y terceros sin intervenir en la custodia de fondos.

Bancos

Custodia, autorización y cumplimiento

Mantienen el control financiero, validan la transacción y conservan la responsabilidad regulatoria.

Telered

APIs, orquestación e interoperabilidad

Estandariza la integración, enruta la operación y habilita la conexión entre participantes.

Empresas / Fintechs

Integración y originación de transacciones

Usan la API para iniciar pagos, recaudos o payouts sobre una sola capa técnica.

Flujo simplificado

De la solicitud de empresa a la confirmación bancaria, con Telered como capa de integración.

  1. 1

    Empresa inicia transacción vía API

  2. 2

    Telered enruta y estandariza

  3. 3

    Banco valida y autoriza

  4. 4

    Se ejecuta y se confirma la operación

6. Plan de Trabajo e Implementación

Despliegue progresivo

Estrategia de Implantación – Despliegue Progresivo

Implementación gradual, priorizando casos de alto volumen y bajo riesgo, para validar el modelo, generar ingresos tempranos y escalar progresivamente.

Principio

No se busca lanzar todo de una vez, sino activar lo suficiente para probar la operación, monetizar temprano y construir efecto red fase a fase.

Fase 1

0–6 meses

Fundacional

Objetivo: Validar el modelo con pocos participantes y casos de uso controlados.

Alcance

  • APIs y SDK base
  • Integración con 2–3 bancos ancla
  • Payouts y recaudos simples

Clientes objetivo

Payroll / BPO y 1–2 fintechs.

Resultado esperado

Validación técnica, primeros ingresos y ajuste del pricing.

Fase 2

6–18 meses

Expansión

Objetivo: Escalar volumen y ampliar casos de uso.

Alcance

  • Más bancos integrados
  • Pagos A2A, QR y links
  • Reporting y conciliación

Clientes objetivo

Utilities, marketplaces y educación.

Resultado esperado

Crecimiento acelerado, consolidación comercial y efecto red inicial.

Fase 3

18–36 meses

Escalamiento

Objetivo: Convertirse en infraestructura estándar del mercado.

Alcance

  • Integración masiva de actores
  • Routing inteligente
  • Interoperabilidad completa

Clientes objetivo

Retail grande, gobierno y plataformas regionales.

Resultado esperado

Alto volumen, ingresos recurrentes relevantes y posición dominante como rail de pagos.

Claves de éxito

1. Empezar por volumen

Payroll y recaudos primero.

2. Bancos ancla

2–3 bancos fuertes al inicio.

3. Monetizar desde fase 1

Aunque sea con bajo volumen.

4. Land & expand

Un cliente puede abrir múltiples productos.

Mensaje ejecutivo

La estrategia propone un despliegue progresivo que minimiza riesgos, asegura adopción temprana y permite escalar el modelo hasta consolidarlo como infraestructura estándar del sistema financiero.
